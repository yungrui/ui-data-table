data-table.js 元件功能展示

一個輕量、零依賴、功能豐富且可高度客製化的純 JavaScript Web Component 資料表格。它內建了類似 Bootstrap 5 的預設樣式，確保在任何專案中都能擁有專業的外觀。

API 總覽

HTML Attributes (透過 <ui-data-table> 標籤設定)

這些屬性提供最直接的設定方式，適合在 HTML 中靜態定義表格行為。

屬性名稱說明預設值範例
data-source-type 指定資料來源的類型。
  • local：表示使用 setData() 傳入的本地陣列，所有排序、分頁、篩選都在前端處理。
  • server：表示將啟用**完整的伺服器端處理**。所有的排序、分頁、篩選操作都會向 fetch-data-url 發送請求，並由伺服器處理。

伺服器端 API 請求參數 (data-source-type="server" 時)

當 data-source-type 為 server 時，元件會自動將以下參數合併到請求中：
  • **分頁**：
    • _page: 當前頁碼 (從 1 開始)。
    • _limit: 每頁顯示的資料筆數。
  • **排序**：
    • _sort: 排序的欄位 (對應 columns 中的 field)。
    • _order: 排序順序 (asc 或 desc)。
  • **篩選**：
    • 對於每個有值的篩選輸入框 (data-filter-field="xxx")，會傳送對應的鍵值對。例如：fieldName=filterValue。
  • **基礎請求資料**：您在 setData 方法中傳入的 requestData 會與上述參數合併。

伺服器端 API **必須**在回應的 HTTP Headers 中包含 X-Total-Count 欄位，以告知元件總資料筆數，否則分頁將無法正常運作。

伺服器端 API 回應內容 (data-source-type="server" 時)

伺服器端 API應回傳一個 JSON 格式的陣列，其中包含當前頁面所顯示的資料。例如：

[
  { "id": 1, "name": "User 1", "email": "user1@example.com" },
  { "id": 2, "name": "User 2", "email": "user2@example.com" }
]

JavaScript Properties & Methods

這些屬性與方法需要透過 JavaScript 在取得元件實體後進行設定或呼叫，適合用於動態定義或修改表格內容。

屬性/方法說明範例
columns (必要) 用於定義表格所有欄位的陣列。每個欄位都是一個物件，可設定以下屬性：
  • field (字串): (必要) 該欄位對應資料來源中的鍵名 (key)。
  • title (字串): 該欄位在表格標頭中顯示的文字。
  • sortable (布林值): 設定為 true 時，該欄位可進行排序。預設為 false。
  • filterable (布林值): 設定為 true 時，該欄位會顯示篩選輸入框。預設為 true (除非全域 filterable 屬性為 false)。
  • render (函式): 一個用於自訂儲存格內容的函式。接收當前行的資料物件作為參數，並可回傳 HTML 字串、HTMLElement 或 DocumentFragment。
  • isCheckbox (布林值): 設定為 true 時，該欄位將渲染為一個核取方塊，用於行選取。預設為 false。
  • cellClass (字串): 可選的 CSS 類別，將會應用於此欄位的每個儲存格 (<td>)。
  • headerClass (字串): 可選的 CSS 類別，將會應用於此欄位的標頭 (<th>)。
table.columns = [
  { field: 'id', title: 'ID', sortable: true },
  { field: 'name', title: '姓名' }
];
setData(data, method, requestData, headers, keepHeaders) (Async) 用於設定表格資料的核心方法。此方法會根據 data-source-type 屬性的值來決定行為。
  • data (陣列 | null):
    • 當 data-source-type 為 'local' (預設) 時，此參數必須是物件陣列。
    • 當 data-source-type 為 'server' 時，此參數通常為 null。
  • method (字串): 當 data-source-type 為 'server' 時，指定 HTTP 請求方法。可選值為 'get' (預設) 或 'post'。
  • requestData (物件 | null):
    • 用於傳遞基礎請求參數。在 server 模式下，此物件中的鍵值對會與動態產生的分頁、排序、篩選參數合併後發送到伺服器。
  • headers (物件 | null):
    • 僅在 data-source-type="server" 時作用。用於設定自定義 HTTP 請求標頭（如：Authorization）。
  • keepHeaders (布林值):
    • (新) 當設為 true 時，傳入的 headers 會與現有的 Headers 合併，而不是完全取代。若傳入的 headers 為 null，則會完整保留現有的 Headers。

回傳：一個 Promise，resolve 時帶入最終渲染後的資料陣列。失敗時會 throw 錯誤。

// 1. 本地資料 (前端處理)
table.setData(myLocalArray);

// 2. 伺服器資料 (GET，無初始參數)
table.setData(null);

// 3. 伺服器資料 (POST 帶有自定義 Header)
table.setData(null, 'post', { search: 'keyword' }, {
  'Authorization': 'Bearer YOUR_TOKEN'
});

// 4. 更新資料但保留現有的 Headers
table.setData(null, 'get', null, null, true);
updateHeaders(headers) (新) 用於更新伺服器端請求的 Headers。
  • 此方法會將傳入的 headers 物件與現有的 Headers 合併。
  • 適合用於更新 Token 等會隨時間變動的認證資訊，而不需要重新設定整個 setData 參數。
table.updateHeaders({
  'Authorization': 'Bearer NEW_TOKEN'
});
Custom Events (新) 元件會拋出以下自定義事件，方便與外部整合：
  • row-click: 點擊資料列時。detail: { rowData, rowId }
  • selection-change: 勾選狀態改變時。detail: { selectedRows }
  • page-change: 換頁或改變每頁筆數時。detail: { pageIndex, pageLength }
  • sort-change: 排序改變時。detail: { field, order }
table.addEventListener('row-click', (e) => {
  console.log('Clicked row:', e.detail.rowData);
});
pageLengthList設定「每頁筆數」下拉選單的選項。是一個數字陣列。
table.pageLengthList = [5, 15, 30];
onBeforeRender 在表格實際繪製到畫面之前執行的回呼函式。
  • 接收參數：當前頁面的資料陣列。
  • 功能：可以直接修改該陣列，或者回傳一個新的陣列作為渲染資料。
table.onBeforeRender = (data) => {
  // 對當前分頁資料做處理
  data.forEach(item => item.name += ' (已處理)');
};
getSelectedRowsData()當有欄位設定為 isCheckbox: true 時，此方法可用於獲取所有被勾選的資料列的完整資料物件陣列。
const selected = table.getSelectedRowsData();
clearState()當 persist-state 功能啟用時，呼叫此方法可清除此表格在 localStorage 中保存的狀態。
table.clearState();
外部樣式設定 (CSS Shadow Parts)

此元件支援使用 ::part() 偽元素來客製化內部的元件樣式，讓您可以覆寫預設的外觀以符合您的設計需求。這允許您在不修改元件內部 CSS 的透過與專案的全局樣式保持一致。

例如，若要改變表格標頭和分頁容器的樣式：


/* 選取 id 為 my-table 的元件 */
#my-table::part(table-header) {
    background-color: #6f42c1; /* 改變標頭背景色 */
    color: white;
}
#my-table::part(pager-container-top) {
    background-color: #f3e5f5; /* 改變上方分頁背景色 */
    padding: 0.5rem;
    border-radius: 0.3rem;
}
#my-table::part(table) {
    border: 2px solid #007bff; /* 改變表格邊框 */
}

所有可用的 part 名稱：

  • pager-container-top: 上方的分頁容器。
  • pager-container-bottom: 下方的分頁容器。
  • table-container: 包覆 <table> 的響應式容器。
  • table: <table> 元素本身。
  • table-header: <thead> (表格標頭) 元素。
  • table-body: <tbody> (表格主體) 元素。
  • table-header-cell: 表格標頭單元格 (<th>)。
  • table-cell: 表格資料單元格 (<td>)。
  • loading-overlay: 載入中遮罩層.
  • loading-spinner: 載入中動畫圖示.

範例 2: 伺服器端處理 (GET)

此表格透過 data-source-type="server" 啟用完整的伺服器端處理。所有排序、分頁、篩選操作都會向 http://localhost:3000/users 發送 GET 請求。請開啟瀏覽器開發者工具的網路 (Network) 頁籤觀察請求。

例如，可以使用 json-server 建立模擬 API，並啟動：json-server --watch users.json --routes routes.json。 其中 `routes.json` 內容類似：`{ "/users": "/users?_embed=posts" }`，確保 `X-Total-Count` 標頭能被正確回傳。
